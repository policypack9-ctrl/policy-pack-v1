# PolicyPack

PolicyPack is a modern SaaS platform built with Next.js (App Router) that generates custom legal documents (Privacy Policy, Terms of Service, Cookie Policy, GDPR Addendums) using AI through OpenRouter.

## Tech Stack

- **Framework**: Next.js 16 (App Router) + React 19
- **Styling**: Tailwind CSS v4, Framer Motion, Base UI, Shadcn
- **Authentication**: NextAuth.js v5 beta (Credentials + Google OAuth)
- **Database**: PostgreSQL via Supabase (w/ Supabase Adapter)
- **AI Generation**: OpenRouter API (Claude 3.7 / Gemini 1.5)
- **Billing**: Paddle Checkout & Webhooks

## Getting Started

### 1. Environment Setup

Copy the example environment file and configure your keys:

```bash
cp .env.example .env.local
```

You will need to set up:
- **Supabase**: Create a new project, copy URL & Keys. Apply the schema from `supabase/schema.sql`.
- **Google Cloud**: Set up an OAuth app for `AUTH_GOOGLE_ID`.
- **OpenRouter**: Get an API key for the document generation engine.
- **Paddle**: Configure sandbox environment, products, and webhook secret.

*(For local DB, you can use the included `docker-compose.yml` to spin up PostgreSQL)*

### 2. Install Dependencies

```bash
npm install
```


### 3. Run the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Architecture Highlights

1. **Two-Stage AI Generation**: Documents are generated by first calling a "Research" model to summarize the latest laws based on user onboarding data, followed by a "Drafting" model to generate the actual Markdown document.
2. **Local First, Database Ready**: Onboarding sessions and generated documents sync seamlessly between `localStorage` and `Supabase` once the user logs in.
3. **Verified Premium Flow**: The app uses Paddle Webhooks to verify transactions before upgrading a user's account to Premium.
